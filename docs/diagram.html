Fiducia.cloud · Architecture

Raft-replicated coordination as a service — locks, semaphores, rate limiting, config KV, leader election, service discovery.

1 · Request path & components

Clients speak HTTP to the edge; the edge picks a healthy region; the regional load balancer routes each request by key to the shard's leader.

2 · Multi-cluster: survive losing 1 of 3

Every shard keeps RF=3 with one replica per cluster, and the brain is a 3-member Raft group (one per cluster). Lose any one cluster → every shard and the control plane keep a 2/3 majority → still serving.
